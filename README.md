# Customer-Behavior-Analysis
Overview
This project demonstrates an end-to-end data analytics workflow, covering data loading, exploratory data analysis (EDA), data cleaning, SQL querying, and business intelligence visualization.
The goal is to extract meaningful insights from raw data and present them clearly through dashboards, reports, and presentations.
This project showcases practical data analyst skills using Python, SQL, and Power BI.

The dataset contains structured business data (CSV format).
Includes both numerical and categorical variables.
Realistic data issues such as:
Missing values
Duplicates
Inconsistent formats
The dataset is used consistently across Python, SQL, and Power BI for analysis.

Prject Process Flow
1️. Data Loading
Loaded the dataset into Python using Pandas.
Imported cleaned data into SQL databases for querying.

2️. Exploratory Data Analysis (EDA)
Analyzed:
Data distribution
Trends and patterns
Outliers
Used visualizations to understand relationships between variables.

3️. Data Cleaning
Handled missing values (mean/median/group-based methods).
Removed duplicates.
Standardized column names and data formats.
Validated data consistency.

4️. SQL Analysis
Loaded data into:
PostgreSQL
Wrote SQL queries for:
Aggregations
Filtering
Joins
Business KPIs

5️. Dashboard Creation
Built an interactive Power BI dashboard including:
KPIs
Trend analysis
Category-wise breakdowns
Filters and slicers

6️. Reporting & Presentation
Created a structured analysis report summarizing insights.
Designed a professional presentation using Gamma, focusing on:
Problem statement
Key findings
Business recommendations
